# 統括PM引き継ぎガイド - 完全版

## 🎯 このドキュメントの目的
統括PMが停止した場合に、新しい引き継ぎ者が即座に同じ認識と実行能力を持てるようにする。

## 📋 現在のシステム概要

### システム構成
```
【階層構造】
ユーザー
 ↓
統括PM（あなたが引き継ぐ役割）
 ↓
サブPM（Task toolで起動）
 ↓
作業者AI（アインシュタインクイズを解く）
```

### 役割定義
- **統括PM**: システム全体の設計、環境構築、サブPM管理、問題解決
- **サブPM**: 複数の作業者AIを管理、実際の作業指示と結果収集
- **作業者AI**: アインシュタインクイズを独立して解答

## 📝 統括PMとしての作業手順

### Phase 1: サブPM起動と指示
```
Task toolを使用し、以下のプロンプトでサブPMを起動：

【X体管理の場合】
あなたは複数のAIにアインシュタインクイズを解かせる司会進行役です。
今回はX体のAIを管理して、それぞれにクイズを解かせてください。
以下の作業を最後まで完遂してください。途中で止まらず、全て実行してください。

【環境確認と準備】
現在地が/mnt/c/Users/user/claude-test-100-v0.2であることを確認し、以下を実行：
1. ls einstein-quiz-answers/ で既存回答を確認
2. 次はAI[開始番号]から[終了番号]のX体を実施することを確認

【X体のAIインスタンス起動と管理】
各AIに対して以下を実行（AI[番号]の順番で）：

AIごとの手順：
1. cd auto-claude-start でスクリプトフォルダに移動
2. powershell.exe -ExecutionPolicy Bypass -File einstein_quiz_launcher.ps1 を実行
3. 「IMPORTANT: Click on the new command prompt window」が出たら3秒以内に新ウィンドウをクリック
4. 「Answer will be saved to: /tmp/einstein_answer_XXXXXX.txt」のタイムスタンプをメモ
5. 5-10分待ってAIが解き終わるのを確認
6. cat /tmp/einstein_answer_[タイムスタンプ].txt で回答確認
7. cp /tmp/einstein_answer_[タイムスタンプ].txt einstein-quiz-answers/AI[番号]-answer.txt で保存

【完了報告】
全X体の処理が終わったら、以下の形式でまとめて報告：

**X体実施完了報告**
- 実施AI: AI[リスト]
- 各AIの結果：
  - AI[番号]: 開始時刻、最終回答（誰が魚を飼っているか）
- 保存ファイル: AI[番号]-answer.txt のリスト
- 全体ステータス: 完了

これら全てを順番に実行し、最後の報告まで完了させてください。
```

## 🚨 重要な技術的知識

### PowerShellスクリプトの仕組み
- `einstein_quiz_launcher.ps1`が新しいClaude AIインスタンスを起動
- 3秒以内のウィンドウクリックが【必須】
- タイムスタンプ付きで`/tmp/`に一時保存
- サブPMが整理された名前でコピー

### ファイル命名規則
- 一時ファイル: `/tmp/einstein_answer_20250802_085935.txt`
- 最終ファイル: `einstein-quiz-answers/AI1-answer.txt`

### エラーハンドリング
- **ウィンドウクリック忘れ**: 最初からやり直し
- **ファイルが見つからない**: フルパスを確認
- **サブPM停止**: Task toolで再起動
